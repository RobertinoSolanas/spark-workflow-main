# Security Policy

## Supported Versions

This section describes which versions of the project currently receive security updates.

| Version | Supported |
|--------|-----------|
| latest | ✅ |
| older versions | ❌ |

Users are encouraged to upgrade to the latest release to receive security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, **please do not report it publicly in issues or discussions**.

Instead, report the vulnerability privately to the maintainers so that it can be investigated and fixed responsibly.

Please include the following information in your report:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Affected versions
- Suggested fix (if available)
- Any relevant logs or screenshots

Providing detailed information helps maintainers resolve the issue faster.

---

## Contact

Security vulnerabilities should be reported directly to the project maintainers.

You can contact the maintainers via:

- Email: planungsbeschleunigung@bmi.bund.de
- Direct contact with a maintainer listed in `MAINTAINERS.md`

If email is used, please include the project name and a short description of the issue in the subject.

---

## Response Process

After receiving a vulnerability report, maintainers will:

1. Acknowledge the report
2. Investigate the issue
3. Develop and test a fix
4. Prepare a security release
5. Publish the fix and notify users

We aim to respond to security reports as quickly as possible.

---

## Responsible Disclosure

We kindly ask researchers and contributors to follow responsible disclosure practices:

- Do not publicly disclose the vulnerability before a fix is available
- Allow maintainers reasonable time to investigate and resolve the issue
- Coordinate disclosure with the maintainers if necessary

Responsible disclosure helps protect users of the project.

---

## Security Updates

Security fixes will be released as part of a new version of the project.

Release notes will include information about the vulnerability and affected versions when appropriate.

---

## Acknowledgements

We appreciate responsible disclosure and thank security researchers who help improve the security of this project.
