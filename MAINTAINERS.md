# Maintainers

This document lists the maintainers of this project and describes their responsibilities.

Maintainers are responsible for reviewing contributions, guiding the development of the project, and ensuring the long-term health of the repository.

# Responsibilities

Maintainers are responsible for:

- Reviewing and approving pull requests
- Ensuring code quality and project standards
- Managing issues and discussions
- Guiding project direction and roadmap
- Maintaining documentation
- Managing releases

Maintainers should aim to review contributions in a timely and constructive manner.

---

# Becoming a Maintainer

Contributors may be invited to become maintainers if they:

- Consistently contribute high-quality code
- Actively participate in reviews and discussions
- Demonstrate good understanding of the project
- Show commitment to the project's long-term development

Maintainers are typically nominated by existing maintainers.

---

# Maintainer Expectations

Maintainers are expected to:

- Act in the best interest of the project
- Be respectful and supportive toward contributors
- Follow the project's Code of Conduct
- Provide constructive feedback
- Encourage community participation

---

# Inactive Maintainers

Maintainers who are inactive for an extended period of time may be moved to an **emeritus** or **inactive** status.

Inactive maintainers may be reactivated upon request.

---

# Security Contact

For reporting security vulnerabilities, please refer to the `SECURITY.md` file.

Do not report security issues in public issue trackers.

---

# Maintainer Changes

Changes to the maintainer list should be made through a pull request updating this file.

Such changes typically require approval from existing maintainers.
