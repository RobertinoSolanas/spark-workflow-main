# Support

Thank you for using this project.

This document describes how to get help, report problems, or ask questions related to the project.

---

## Getting Help

If you need help using the project, please try the following steps:

1. Check the project documentation (README or other docs).
2. Search existing issues to see if your question has already been answered.
3. Open a new issue if the problem has not been reported yet.

Providing clear information helps maintainers and contributors assist you more effectively.

---

## Asking Questions

If you have a general question about the project, you can open an issue in the repository.

Please include:

- A clear description of your question
- Relevant context about what you are trying to achieve
- Any steps you have already tried

---

## Reporting Bugs

If you encounter a bug, please open an issue and include:

- A description of the problem
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Environment details (OS, runtime, version)
- Logs or screenshots if available

Example:

Environment:
- OS: Ubuntu 22.04
- Runtime: Node 20
- Project version: v1.2.0

Steps to reproduce:

1. Run command X
2. Perform action Y
3. Observe the issue

Expected result:
The operation succeeds.

Actual result:
The application crashes.

---

## Feature Requests

If you have an idea for a new feature or improvement, feel free to open an issue.

Please describe:

- The problem you are trying to solve
- The proposed solution
- Possible alternatives

Feature requests are welcome, but implementation depends on project priorities and maintainer availability.

---

## Security Issues

If you discover a security vulnerability, **do not open a public issue**.

Please follow the instructions in `SECURITY.md` to report the vulnerability privately.

---

## Contribution

If you would like to contribute to the project, please read the guidelines in `CONTRIBUTING.md`.

Contributions of all sizes are welcome.
